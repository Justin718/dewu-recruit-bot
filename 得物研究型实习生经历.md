# 得物研究型实习生

## 2026-03 ~ 2026-06 &nbsp;&nbsp;&nbsp;&nbsp; **得物App** &nbsp;&nbsp;&nbsp;&nbsp; **算法策略部-推荐系统方向**

独立负责商品推荐场景下的用户行为建模与序列化召回优化，处理亿级用户-商品交互数据，支撑核心推荐链路CTR提升8%+，服务DAU千万级用户的个性化推荐需求

### • 业务背景与痛点
得物作为头部潮流电商平台，用户对新品、限量款、联名款等商品的发现效率直接影响GMV与留存。当前推荐系统存在：冷启动商品曝光不足导致长尾库存积压；用户兴趣漂移捕捉滞后；跨品类（鞋服配饰）兴趣迁移建模不充分等问题。需要构建更精细的用户理解模型与动态召回机制，同时兼顾业务指标（点击率、转化率、加购率）与用户体验（多样性、惊喜度）。

### • 核心工作内容
1. **用户长短期兴趣建模**：基于Transformer架构设计多尺度兴趣提取模块，分别捕获用户近期实时兴趣（短期窗口7天）与长期偏好画像（90天历史聚合），通过注意力门控机制自适应融合双塔表征输入下游排序模型
2. **序列化召回优化**：引入SASRec/BERT4Rec等序列推荐范式，利用用户最近50次交互行为预测下一候选集，结合Item-CF协同信号进行多路召回融合，新增路权占比达25%
3. **商品知识图谱增强**：爬取并清洗商品属性数据（品牌、品类、风格标签、发售时间、价格带），构建同款/相似款/替代品关系边，用于推理类query的语义召回与解释性推荐
4. **冷启动策略设计**：针对新上架潮品，基于内容特征（图片CNN Embedding+文本属性）计算与历史热门品的相似度，采用Bandit算法（LinUCB）进行探索-利用均衡分配初始流量，加速冷品积累正向反馈
5. **离线评估体系搭建**：构建A/B测试平台对接线上实验，设计分层抽样与SRM检验流程，确保实验结论统计显著；同时完善离线指标（NDCG@K、HitRate、Coverage）与业务指标的对齐验证

### • 技术栈
PyTorch, Transformers, TensorFlow Serving, Spark/Flink (特征工程), Redis (在线特征), Hive/ClickHouse (数仓), Airflow (调度), Docker/K8s (部署), SQL, Python

### • 已有信息闭环分析
结合得物公开技术博客、推荐系统顶会论文（RecSys/KDD/WWW），从用户行为漏斗（浏览→点击→加购→下单→复购）出发，拆解各环节优化空间：曝光层侧重多样性与公平性保障，排序层强化即时反馈信号权重，重排层加入业务约束（库存、利润率、品牌均衡）。项目产出直接应用于得物App首页猜你喜欢、详情页相关推荐、购物车搭配推荐等核心场域，形成**数据→模型→实验→上线→监控**完整闭环。
